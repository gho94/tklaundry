## 1-1. 회원

| # | 확인 | 기대 |
|---|------|------|
| 1-1 | 로그인 | 빈 값·틀린 비번 차단 |
| 1-2 | 로그인 성공 | 메인 이동, 상단 사용자명 표시 |
| 1-3 | authProvider | `userId`·`userName` 저장 |
| 1-4 | 자동 로그인 | 체크 후 로그인 → 앱 재실행 시 메인 진입 |
| 1-5 | 신규 가입 | 로그인 화면 가입 → 별도 로그인 없이 메인 진입 |
| 1-6 | 회원 CRUD | 목록·등록·수정·삭제·`UseYn` 레거시와 DB 동일 |
| 1-7 | 본인 삭제 | `409 CONFLICT` |

**API 수동 확인**

```powershell
# 목록
curl http://localhost:8080/api/members

# 아이디 중복 확인 (true = 이미 존재)
curl "http://localhost:8080/api/members/exists?userId=test01"

# 회원가입 (201 + member body)
curl -X POST http://localhost:8080/api/auth/register -H "Content-Type: application/json" -d "{\"userId\":\"test01\",\"password\":\"pw\",\"userName\":\"테스트\",\"useYn\":\"Y\"}"

# 상세
curl http://localhost:8080/api/members/test01

# 수정 (password 생략 시 기존 비밀번호 유지, 204)
curl -X PUT http://localhost:8080/api/members/test01 -H "Content-Type: application/json" -d "{\"userName\":\"테스트수정\",\"useYn\":\"Y\"}"

# 삭제 (204)
curl -X DELETE http://localhost:8080/api/members/test01
```

레거시 `FrmMember`와 DB `ComMember` 행이 동일한지 대조한다.

## 1-2. 공통코드

| # | 확인 | 기대 |
|---|------|------|
| 2-1 | login 캐시 | login 응답 `codes` · `codeProvider` 적재 · 로그아웃 초기화 |
| 2-2 | 목록 API | `GET /api/codes` flat JSON |
| 2-3 | 트리 UI | ROOT 하위 구조 일치 (설정→코드) |
| 2-4 | 선택 상세 | `codeProvider` flat — ID·코드명·상위 ID·등급 |
| 2-5 | 등록 | API·UI — 최상위·하위 채번 · 트리 갱신 |
| 2-6 | 수정·삭제 | API·UI — 하위 재귀 삭제 · CRUD 후 `codeProvider` 갱신 |
| 2-7 | 캐시 반영 | 코드 화면 진입 시 캐시 있으면 별도 목록 조회 없음 |

**API 수동 확인**

```powershell
# 로그인 (200, user + codes)
curl -X POST http://localhost:8080/api/auth/login -H "Content-Type: application/json" -d "{\"userId\":\"admin\",\"password\":\"...\"}"

# 목록 (flat, 캐시 갱신용)
curl http://localhost:8080/api/codes

# 최상위 등록 (201)
curl -X POST http://localhost:8080/api/codes -H "Content-Type: application/json" -d "{\"pCodeId\":\"Root\",\"codeName\":\"테스트분류\"}"

# 하위 등록 (201)
curl -X POST http://localhost:8080/api/codes -H "Content-Type: application/json" -d "{\"pCodeId\":\"A00001\",\"codeName\":\"하위코드\"}"

# 수정 (204)
curl -X PUT http://localhost:8080/api/codes/A00001 -H "Content-Type: application/json" -d "{\"codeName\":\"변경된 코드명\"}"

# 삭제 (204, 하위 포함)
curl -X DELETE http://localhost:8080/api/codes/A00001

# 필수값 누락 (400 VALIDATION_ERROR)
curl -X POST http://localhost:8080/api/codes -H "Content-Type: application/json" -d "{\"pCodeId\":\"Root\"}"
```

응답 예 (login): `{ "user": { "userId": "admin", "userName": "관리자" }, "codes": [ ... ] }`

응답 예 (등록 201): `{ "codeId": "A10004", "pCodeId": "A00001", "codeName": "하위코드" }`

**앱 확인**

- [ ] **로그인** → 코드 화면 즉시 트리 표시 (login 캐시)
- [ ] **로그아웃** → `codeProvider` 초기화
- [ ] **최상위·하위 추가** → 트리 갱신
- [ ] **수정** → flat 캐시 기준 다이얼로그 · 저장 후 트리 반영
- [ ] **삭제** → 하위 포함 확인 · subtree 제거
- [ ] 실패 시 API 메시지·`traceId` 표시

레거시 코드관리(`FrmBaseData`)와 DB `ComBaseData` 행·트리 구조가 동일한지 대조한다.
