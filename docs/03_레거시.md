# 레거시 참고 요약

> C# WinForms + DevExpress. DB 직접 접속. API 없음. 매장 PC 1명 사용.

**소스 위치**: 차세대 레포 **밖** 형제 폴더 `TKLaundry/` (비교용, bin/obj 제외)

---

## DB 핵심 테이블

| 테이블 | 용도 | 주요 컬럼 |
|--------|------|-----------|
| `ComMember` | 로그인·회원 | `UserID`, `UserPW`, `UserName`, `UseYn` |
| `ComBaseData` | 공통코드 | `CodeID`, `PCodeID`, `CodeName` |
| `ComCustomer` | 고객 | `CustCode`, `CustName`, `AptCode`, `BuildingCode`, `FloorCode`, `RoomCode`, `CustPhone` |
| `ComProduct` | 제품·단가 | `ProductCode`, `ProductName`, `Price`, `ProcessCode` |
| `OrderMaster` | 접수 헤더 | `OrderNo`, `OrderDate`, `CustCode`, `Status`, `Cost` |
| `OrderDetail` | 접수 상세 | `OrderNo`, `OrderSeq`, `ProductCode`, `Qty`, `Cost` |
| `DeliveryMaster/Detail` | 출고 | 접수 `OrderNo` 연결 |
| `SalesMaster/Detail` | 매출·수납 | 접수 연결 |
| `AutoNumber` | 번호 채번 | `DocName`, `DocDate`, `Seq` |

## 차세대 구현 순서

> **접수·출고·매출은 아래 4개 마스터가 레거시와 같아진 뒤** 시작.

| 순서 | 영역 | 테이블 | 레거시 화면 |
|:----:|------|--------|-------------|
| 1 | 회원 | `ComMember` | 로그인, 회원관리 |
| 2 | 공통코드 | `ComBaseData` | 코드관리 |
| 3 | 고객 | `ComCustomer` | 고객관리 |
| 4 | 제품 | `ComProduct` | 제품관리 |
| 5 | 접수 | `OrderMaster/Detail` | 접수 |
| 6 | 출고 | `Delivery*` | 출고 |
| 7 | 매출 | `Sales*` | 수납·매출 |

## 공통코드

- DB: `PCodeID`로 부모-자식 (트리 구조)
- **레거시 앱**: `Dictionary<string, TblComBaseData>` — 키 = `CodeID`
- 콤보: `PCodeID`로 자식 필터
- 차세대 API: flat list → 앱이 Map으로 변환

| 코드 | 의미 |
|------|------|
| `B20001` | 일반 |
| `B20002` | 선불 |
| `B20005` | 외상 |

---

## 고객 (`FrmCustomer` / `FrmCustomerInsert`)

- 목록: 상단 **아파트 콤보** 필터 (`전체` / 특정 아파트 / `기타`) — 텍스트 `q` 검색 없음
- 정렬: `AptCode, BuildingCode, FloorCode, RoomCode, CustName`
- 등록·수정: 동일 다이얼로그 — 콤보 연쇄(아파트→동→층→호), 이름 자동 조합, 전화번호
- 수정: 그리드 **더블클릭** → 행 값으로 다이얼로그 (단건 SELECT 없음)
- 중복: **신규 등록** 시 4콤보 모두 선택된 경우만 위치 중복 검사
- 채번: `C` + 4자리 (`MAX(CustCode)+1`, `AutoNumber` 미사용)
- 삭제: `DELETE` (확인 메시지 후)

## 접수 (2단계 때 참고)

**번호**: `O` + yyMMdd + `-` + 3자리 (`AutoNumber`, `DocName='접수'`)

**상태 흐름**:
```
접수대기 → 작업중 → 완료 → 출고대기 → 출고완료
         ↘ 취소
```

**2단계 전에 레거시 소스로 확인할 것**:
- 완료 = 출고대기인지
- 취소 가능 시점
- 출고 시 미납 처리
- 대조 기준 날짜 (`OrderDate`?)

---

## 레거시 특징

- 삭제: `DELETE` (소프트 삭제 없음)
- SP/Trigger 없음
- 금액·수량: `int`
